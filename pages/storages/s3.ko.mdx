# S3 저장소

### 개요

S3 저장소는 S3 호환 스토리지에 데이터를 저장합니다. 이 저장소는 AWS S3, [MinIO](https://min.io/) 등의 호환 스토리지를 사용할 수 있습니다.

### 설정 데이터 구조

설정 데이터는 다음과 같은 구조를 가집니다.

```json
{
    "bucketName": "bucket-name",
    "region": "region",
    "endpoint": "endpoint",
    "credentials": {
        "accessKeyId": "<ACCESS_KEY_ID>",
        "secretAccessKey": "<SECRET_ACCESS_KEY>"
    }
}
```

#### `bucketName` 필드 [#bucket-name]

| 항목     | 내용      |
|--------|---------|
| 데이터 형식 | String |
| 필수 여부  | ✅       |

데이터를 저장할 버킷의 이름입니다.

#### `region` 필드 [#region]

| 항목     | 내용      |
|--------|---------|
| 데이터 형식 | String |
| 필수 여부  | ✅       |

버킷이 위치한 리전의 이름입니다.

#### `endpoint` 필드 [#endpoint]

| 항목     | 내용      |
|--------|---------|
| 데이터 형식 | String |
| 필수 여부  | ❌       |

버킷에 접근할 수 있는 엔드포인트의 URL입니다. 이 값이 설정되지 않으면 AWS S3의 기본 엔드포인트가 사용됩니다.

#### `credentials` 필드 [#credentials]

| 항목     | 내용      |
|--------|---------|
| 데이터 형식 | Object |
| 필수 여부  | ❌       |

버킷에 접근할 수 있는 자격 증명입니다. 이 값이 설정되지 않으면 기본 자격 증명이 사용됩니다.

### 예제

```json
{
    "bucketName": "cabinetjs-data",
    "region": "ap-northeast-2",
    "endpoint": "https://endpoint",
    "credentials": {
        "accessKeyId": "<ACCESS_KEY_ID>",
        "secretAccessKey": "<SECRET_ACCESS_KEY>"
    }
}
```

위 예제는 제공된 자격 증명을 사용하여  `ap-northeast-2` 리전의 `cabinetjs-data`라는 이름의 버킷에 `https://endpoint`를 통해 접근,
저장하는 S3 호환 스토리지를 생성합니다.
